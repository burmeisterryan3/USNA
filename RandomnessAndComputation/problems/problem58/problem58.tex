The expected worst case cost of insertion or search for a randomized
binary search tree is $O(logn)$ as the height is expected to be somewhere
on the order of the $lg$ of the number of inputs.  Search can be performed
in the same manner as is done for an AVL or 2-3-4 tree.  The algorithm
is shown below.


\begin{algorithm}
\begin{algorithmic}[1]
    \State {Start at root node}
    \While {node $!=$ null and node.value $!=$ searchValue}
        \If {searchValue $>$ node.value}
            \State {node = node.rightChild}
        \ElsIf {searchValue $<$ node.value}
            \State {node = node.leftChild}
        \Else
            \State {return node.value}
        \EndIf
    \EndWhile
    \State {print ``Value not in RBST''}
\end{algorithmic}
\end{algorithm}
